# 問 1
あなたのチームは AWS を使い様々なサービスを運用しています。あなたはインフラ・セキュリティ担当として AWS の設定を定期的にチェックする仕組みを考えています。そのために AWS Config に着目してみました。

要件 1, 要件 2 の様に検知対象とするリソースを定めました。
AWS Config によるリソースの設定スナップショットを読み込み、検知対象の`configurationItems`数をカウントし、標準出力するプログラムを作成してください。

## 要件 1
本番環境用の Amazon EC2 インスタンス（以下 EC2 インスタンス）において、バーストパフォーマンスインスタンスの利用を禁止したいです。
上記に違反する EC2 インスタンスを検知・違反対象としてカウントする事とします。

EC2 インスタンスに `env` というタグを設定することによって本番環境用 EC2 インスタンスと検証環境用 EC2 インスタンスを区別しています。
その値が `prd` であれば本番環境用である事を示し、`stg` であれば検証環境用である事を示しています。
（本番環境用のタグ、検品環境用のタグのどちらも付いてない EC2 インスタンスは検知対象外としてください）

## 要件 2
EC2 インスタンスへの SSH 接続（port 22）を、踏み台 EC2 インスタンスからのみに制限したいです。
上記に違反するセキュリティグループを検知・違反対象としてカウントする事とします。

踏み台 EC2 インスタンスと接続先の EC2 インスタンスは同じ VPC 内に存在しているものとします。
また、踏み台インスタンスは port 22 以外で SSH 接続を受ける設定にしているため今回考慮外としてください。

|  踏み台 EC2 インスタンスのプライベート IP アドレス |
| ---- |
| 10.10.2.100 |
| 10.10.3.100 |

## 出力形式
出力する形式は`N1,N2`とし、これを標準出力する様にしてください。
N1,N2 は、それぞれ 要件1,要件2 に違反しているリソースの数とします。

## サンプルファイル
`data/config.json`には採点で使われるファイルと同じ形式のサンプルを配置してあります。
**フィールドに含まれる値やファイルのサイズは採点で使われるものとは異なっています。**
採点時には、同じパスのファイルが採点用のものに置き換えられた上でプログラムが実行されます。
サンプルファイルを正しいプログラムで実行した場合は標準出力に`1,1`と出力されます。

## 提出物
* 作成したプログラムをGitHubにプッシュしてください。
* このREADMEが置かれているディレクトリをカレントディレクトリとして、 `./run.sh` で起動できるようにしてください。
* プログラムは以下のパフォーマンス要件を満たすようにしてください。
    * 利用可能メモリ: `1GB`
    * 実行時間: `10s`

## 参考実装
JSON ファイルを読んで解答の形式のみを満たした値を出力するだけのプログラムを参考実装として公開しています。
ローカル環境で以下のようにして動かすことが出来ます。

```
$ ./run.sh
```

## フィードバック
採点時に各テストケースにおいて以下のフィードバック文のいずれかが返されます。

* `実行が完了しました`
    * プログラムが正常に出力を行なったことを表します。
    * **出力内容が正しいとは限らない** ので注意してください。（出力内容の正誤はフィードバックには含まれません。）
* `実行時間が制限を超過しました`
    * 処理時間がパフォーマンス要件の制限を超えたことを表します。
* `使用メモリ量が制限を超過しました`
    * 使用メモリ量がパフォーマンス要件の制限を超えたことを表します。
* `ビルドに失敗しました`
    * プログラムのビルドに失敗したことを表します。
    * `build.sh` の内容が正しいことを確認してください。
* `実行が正常に完了しませんでした`
    * プログラムが正常終了しなかったことを表します。
    * プログラムコードおよび `run.sh` の内容が正しいことを確認してください。
* `サンプルのままの提出です`
  * 用意されたサンプルコードや解答例がそのまま提出されたことを表します。
  * なおこのフィードバックは Q-AWSCloudGovernance-Entry 全体で共通であり、no1, no2 のいずれかに変更を加えていればこのフィードバックは返されません。
* `解答ファイルが提出されていません`
   * `run.sh`が提出されていないことを表します。

その他、採点システム上の問題が生じた場合は下記のフィードバック文が返されます。お問い合わせ先にご連絡ください。

* `採点システムにエラーが発生しました`
  * システムのエラーです。お手数ですが本スキルチェックの案内メールに記載のお問い合わせ先にご連絡ください。
